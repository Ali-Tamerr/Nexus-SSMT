# Security Policy

## Supported Versions

We only support security updates for the current major version of Nexus.

| Version | Supported          |
| ------- | ------------------ |
| v0.1.x  | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a potential security vulnerability in our project, **please do not report it publicly via a GitHub issue**.

Instead, please send an email with a detailed description to [security@nexusgraph.com](mailto:security@nexusgraph.com) or contact the maintaining team directly.

### What to include in your report:

- A clear, concise description of the potential vulnerability.
- Steps to reproduce it (POC scripts are welcome).
- Any potential impact you already identified.

### Our Commitment

We will:

- Respond to your report within a reasonable timeframe (typically within 48-72 hours).
- Keep you informed of our progress.
- Publicly credit you for your find if you wish, once the issue is resolved.

Thank you for helping us keep Nexus secure for all students!
